# アーキテクチャ概要

このドキュメントでは、電脳ちゃんDiscordボットプロジェクトの全体構造と設計方針について記述します。

## 目的
このボットは、チャット機能・予定通知などの機能を提供することで、サークル活動の効率化・活性化を支援することを目的としています。

## コードマップ

プロジェクトを構成する主要ディレクトリおよびファイルの役割です。

### main.py
エントリーポイント。Botのインスタンスの生成や初期化処理。
### bot.py
Botクラスの定義。ボットのライフサイクルやDiscord Botの拡張機能（Cog）の管理。
### cogs/
Discord Botの拡張機能（Cog）を格納。
### cogs/chat
LLMを利用したチャット機能の実装。
### cogs/schednotifier
Googleカレンダーに登録した予定の通知機能の実装。
### core/
機能ごとのロジック部品を格納。
### core/chat
チャット機能に関連したロジック部品を格納。会話履歴の管理、要約処理など。

## モジュールの簡易関係図
```text
       ユーザー
          │
          ▼
    Discord Message
          │
          ▼
     ┌───────────┐
     │  Cog層     │ (chat, schedなど)
     └───────────┘
          │
          ▼
     ┌───────────┐
     │  core/    │ (chat_instance,history, encryptorなど)
     └───────────┘
          │
          ▼
     外部API (Google Calendar / Gemini)
```

## 主要技術
- **Discord API**：Botのインタフェース。pycordを使用。
- **LangChain**：会話履歴付きLLM応答の生成。
- **Gemini API**：Google Generative AIによる応答と要約。
- **Google Calendar API**：スケジュール取得。
- **SQLite**：LLMとの会話履歴の保存。
- **Pydantic**：設定モデルのバリデーション。
- **Sphinx**：ドキュメント生成。


設計方針
--------

- モジュール単位で明確に分離し、拡張性を高める
- 学内環境で動作することを前提に、シンプルで保守性の高い構成とする
- トークン使用量の記録によって、将来的なコスト最適化を可能にする
- 会話履歴の要約によって、DBサイズを抑制する
